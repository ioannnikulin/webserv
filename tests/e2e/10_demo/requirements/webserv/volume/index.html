üìù Project description

A web server is a program that receives HTTP requests from clients ‚Äî usually browsers ‚Äî and sends back HTTP responses, often including web content such as HTML pages, images, or other resources.

HTTP stands for HyperText Transfer Protocol. It is the foundational protocol used for transmitting data on the World Wide Web. HTTP defines how messages are formatted and transmitted, and how web servers and browsers should respond to various commands.

There have been several versions of HTTP over the years. The most widely used versions are HTTP/1.0, HTTP/1.1 and HTTP/2. Each version introduces improvements in performance, security, and capabilities. This demo focuses on HTTP/1.0, which is the simplest version and serves as a solid foundation for understanding the protocol.

Furthermore, it explores the networking aspects of web servers, including how they handle connections, manage resources, and communicate with clients. This includes understanding sockets, ports, and the TCP/IP protocol suite that underpins internet communication.

An HTTP request is a message sent by a client to a server, asking for a resource or performing an action on the server.

An HTTP response is a message sent by the server back to the client after receiving and processing an HTTP request. It includes a status line, headers, and an optional body containing the requested resource or error information.

A request identifies a method (sometimes informally called verb) to classify the desired action to be performed on a resource.

This demo covers the core HTTP methods: GET, POST, and DELETE.

üì¶ GET method

A GET request is used to retrieve data from a server. It should not change server state and is commonly used for fetching pages, files, or other readable resources.

A POST request is used to send data to a server to create or process a resource. It often changes server state, such as uploading a file or submitting a form.

curl command to send form-encoded data:

curl command to send file:

curl -i -X POST --data-binary @[/path/to/file] localhost:8888/folder/[filename.ext]

üìç .conf file and available routes

This section demonstrates each location block of the server.

  • GET / root 
    
location / {
	methods GET;
	root tests/e2e/10_demo/requirements/webserv/volume;
	index index.html;
}
  • GET /listme autoindex enabled: will show index if file exists OR throw 403 or 404 
    
location /listme {
	methods GET;
	root tests/e2e/10_demo/requirements/webserv/volume;
	autoindex on;
}
  • GET /dontlistme autoindex disabled: will show index if file exists OR throw 403 or 404 
    
location /dontlistme {
	methods GET;
	root tests/e2e/10_demo/requirements/webserv/volume/dontlistme;
}
  • GET /folder endpoint that allows GET, POST, DELETE methods 
    
location /folder {
	methods GET POST DELETE;
	root tests/e2e/10_demo/requirements/webserv/volume/folder;
	index index.html;
	autoindex on;
	client_max_body_size 1M;
	upload on tests/e2e/10_demo/requirements/webserv/volume/folder;
}
  • /internal_redirect Internal redirect to a page on our server (status 301) 
    
location /internal_redirect {
	return http://127.0.0.1:8888/listme;
}
  • /relative_redirect Relative internal redirect to a page on our server (status 301) 
    
location /relative_redirect {
	return /listme;
}
  • /nowhere Internal redirect to a non-existing location (status 404) 
    
location /nowhere {
	return /doesntexist;
}
